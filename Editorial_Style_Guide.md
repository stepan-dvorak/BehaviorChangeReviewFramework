# Editorial Style Guide

## Purpose
Defines the editorial conventions for every repository document.

## Writing Principles
- Technical, analytical and evidence-based.
- Prefer ISO terminology, then Microsoft terminology, then repository terminology.
- Separate Microsoft Position, Empirical Evidence, Research Interpretation and Framework Implication.
- Avoid marketing language and persuasive writing.

## Standard Document Structure
1. Purpose
2. Scope
3. Method
4. Analysis
5. Findings
6. Threats to Validity
7. Conclusions
8. References

## Review Checklist
- Consistent terminology
- No duplicated text
- Evidence separated from interpretation
- Complete references
- Stable repository voice
